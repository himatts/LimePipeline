# ADR-0001 — Nomenclaturas e invariantes

Estado: Aprobado

Contexto: El proyecto necesita consistencia en nombres de archivos, escenas y colecciones para automatizar flujos.

Decisión:
- Proyecto raíz sigue `^[A-Z]{2}-\d{5}\s+(.+)$`.
- Revisión con una letra A–Z.
- PV/REND/SB/ANIM requieren SC (1–999); si no hay `free_scene_numbering`, `SC` múltiplo de `scene_step`.
- Panels/Operators en inglés para docstrings, UI en inglés (compatible i18n futuro); documentación en Español.

Consecuencias:
- Validaciones centralizadas en `core/validate*.py`.
- Operadores deben delegar lógica a `core/` o `scene/` y mantener feedback claro.

Referencias: ver `ARCHITECTURE.md` y reglas de agentes en `AGENTS.md`.
